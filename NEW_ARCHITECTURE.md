# RGB × WDK — Consumption under the new agnostic architecture

As of Tether's `@tetherto/pear-wrk-wdk@1.0.0-beta.8` release, the
worklet RPC layer no longer exposes chain-specific HRPC methods. Any
WDK-compatible wallet package is driven through a single generic
`callMethod` HRPC call, with the wallet managers injected at bundle
time by [`@tetherto/wdk-worklet-bundler`](https://github.com/tetherto/wdk-worklet-bundler).

This document shows how an app consumes `@utexo/wdk-wallet-rgb`
under that model. For the full method surface, see
[`wdk-wallet-rgb/ARCHITECTURE.md`](https://github.com/UTEXO-Protocol/wdk-wallet-rgb/blob/main/ARCHITECTURE.md).

## Integration in 4 steps

### 1. Declare RGB in `wdk.config.js`

```js
// app-root/wdk.config.js
module.exports = {
  networks: {
    rgb:     { package: '@utexo/wdk-wallet-rgb' },
    bitcoin: { package: '@tetherto/wdk-wallet-btc' }
    // …other chains your app supports
  }
}
```

### 2. Install deps + generate the worklet bundle

```bash
npm i \
  @tetherto/wdk@^1.0.0-beta.7 \
  @tetherto/wdk-wallet@^1.0.0-beta.7 \
  @tetherto/wdk-react-native-core@^1.0.0-beta.8 \
  @tetherto/wdk-secret-manager@^1.0.0-beta.3 \
  @tetherto/wdk-wallet-btc@^1.0.0-beta.8 \
  github:Jainakin/wdk-wallet-rgb#80995f9

npm i -D @tetherto/wdk-worklet-bundler@^1.0.0-beta.3

npx wdk-worklet-bundler generate --install
```

The bundler writes `.wdk-bundle/wdk-worklet.bundle.js` — one multi-platform
artifact containing iOS `.framework` and Android `.so` native addons.

### 3. Mount `<WdkAppProvider>`

```tsx
// app/App.tsx
import { WdkAppProvider } from '@tetherto/wdk-react-native-core'
import bundle from './.wdk-bundle/wdk-worklet.bundle.js'

export default function App() {
  return (
    <WdkAppProvider
      bundle={{ bundle }}
      wdkConfigs={{
        networks: {
          rgb: {
            blockchain: 'rgb',
            config: {
              network: process.env.EXPO_PUBLIC_RGB_NETWORK ?? 'testnet',
              indexerUrl: process.env.EXPO_PUBLIC_RGB_INDEXER_URL ?? 'ssl://electrum.iriswallet.com:50013',
              transportEndpoint: process.env.EXPO_PUBLIC_RGB_TRANSPORT_ENDPOINT ?? 'rpcs://rgb-proxy-testnet3.utexo.com/json-rpc'
            }
          },
          bitcoin: {
            blockchain: 'bitcoin',
            config: { /* your BTC config */ }
          }
        }
      }}
    >
      <YourApp />
    </WdkAppProvider>
  )
}
```

### 4. Use RGB from any component

```tsx
import { useAccount } from '@tetherto/wdk-react-native-core'

function RgbScreen() {
  const rgb = useAccount({ network: 'rgb', accountIndex: 0 })
  const ext = rgb.extension()   // proxy for any WalletAccountRgb method

  return (
    <>
      <Text>Address: {rgb.address}</Text>
      <Button title="Issue NIA" onPress={async () => {
        const res = await ext.issueAssetNia({ ticker: 'TST', name: 'Test', precision: 0, amounts: [1000] })
        console.log(res.assetId)
      }}/>
      <Button title="Blind receive" onPress={async () => {
        const { invoice } = await ext.receiveAsset({ witness: false, amount: 100 })
        console.log(invoice)
      }}/>
      <Button title="Send RGB" onPress={async () => {
        const { hash, fee } = await ext.transfer({ token: assetId, recipient: invoice, amount: 100, feeRate: 2 })
      }}/>
    </>
  )
}
```

All RGB flows from the [method surface
table](https://github.com/UTEXO-Protocol/wdk-wallet-rgb/blob/main/ARCHITECTURE.md#method-surface)
work through the same `ext.methodName(...)` proxy.

## What changed from the old approach

| Before (this repo's legacy `setup.sh`) | Now |
|---|---|
| `pear-wrk-wdk` forked + 20 RGB-specific HRPC methods added | `pear-wrk-wdk` unchanged, agnostic |
| Bundle pre-built and committed to each package | Bundle generated by `wdk-worklet-bundler` at app-build time |
| `WDKService.wdkManager.rgbSend({...})` | `useAccount({network:'rgb'}).extension().transfer({...})` |
| Per-chain handler rewrites required in 3 repos for each new method | Add the method to `WalletAccountRgb`; no repo changes outside `@utexo/wdk-wallet-rgb` |

## Regtest testing (local)

Bring up the Docker regtest stack once (bitcoind + electrs + rgb-proxy):

```bash
cd rgb-lib-nodejs/rgb-lib/tests
docker compose up -d
```

Point the app's `.env` at it (iOS simulator uses `localhost`, Android
emulator uses `10.0.2.2`):

```bash
EXPO_PUBLIC_RGB_NETWORK=regtest
EXPO_PUBLIC_RGB_INDEXER_URL=tcp://localhost:50001       # or tcp://10.0.2.2:50001 for Android
EXPO_PUBLIC_RGB_TRANSPORT_ENDPOINT=rpc://localhost:3000/json-rpc
```

Mine blocks when needed:
```bash
docker exec tests-bitcoind-1 bitcoin-cli -regtest -rpcuser=user -rpcpassword=default_password -rpcwallet=miner -generate 1
```
